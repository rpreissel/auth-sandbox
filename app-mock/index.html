⚙️  Konfiguration
Device ID identifiziert dieses Gerät eindeutig beim Backend.
1 Gerätebindung (Device Registration) POST /api/v1/devices/register

Beim ersten Start generiert die App ein RSA-Schlüsselpaar. Der Public Key wird zusammen mit der Device ID, der vorbelgten User-ID, dem Namen und dem Freischaltcode beim Backend registriert. Der Private Key verbleibt im Keystore des Geräts (simuliert durch crypto.subtle).

2 Login: Challenge anfordern & signieren POST /api/v1/auth/login/start

4 Signatur verifizieren & OIDC Tokens empfangen POST /api/v1/auth/login/verify

Das Backend prüft die Signatur, stellt ein kurzlebiges Device JWT aus und tauscht es über den Keycloak Authorization Code Flow gegen vollständige OIDC Tokens.
